This extension searches and returns pages based on the query entered in the browser's address bar.
For the extension to recognize a query, it must start with the 'h ' string (an 'h' letter followed by a whitespace)

General query form:
h searchTerms... tags... dates... afterFilters... sortingRules...


where:
ArgumentsMeaning
searchTermsStrings which must exist within a page's title or URL
tagsUser-defined and assigned to pages tags
datesDate arguments of special format specifying search timeframes
afterFiltersFilters removing unwanted entries from the resulting set
sortRulesSorting rules to be applied on the resulting set

Any number of arguments of any type can be included in a query, or none at all. The only important thing is their order if they are present.

Search terms

String, words, letters that must exist within a page's title or URL

Example queryExplanation
h wordFind any pages containing 'word' in either URL or title
h word1 word2Find any pages containing 'word1' AND 'word2' in either URL or title

Tags

Tags are special entities assignable by the user to pages for easiness of later finding those pages.
After a tag is assigned to a page through the extensions' popup, pages with such a tag can be found using a tag argument in a query.
Tagged pages will be found through a tag even if they are deleted from history.

Tag argument form:
!tagName

Tag word must be prefixed with the exclamation mark (!)

Example queryExplanation
h !tagWordFind any pages having 'tagWord' assigned to them
h !tag1 !tag2Find any pages having 'tag1' AND 'tag2' tags assigned to them

Dates

Dates limit resulting set to desired time borders.

General date argument form:
[+-=]dd/mm/yyyy/hh:mm

where one of -, +, = signs must be included at the start of the date argument depending on the desired result:

SignExplanation
+after and including date
-before and including date
=exact date

then the sign is followed by one or more date segments specifying the necessary precision of a time frame:

SegmentDescriptionExplanation
dd/one or two digits followed by a slasha number of the month
/mm/a slash followed by one or two digits followed by another slasha number of the month
/yyyy/a slash followed by four digits followed by another slasha year
hh:two digits followed by a colonhours
:mma colon followed by two digitsminutes

Omitted date segments will be assumed to be of the current time (the current year, the current month, etc.).
Multiple date options can be added and combined to specify multiple time spans.

Example queryExplanation
h =/2018/Find pages that were visited in 2018
h =/4/2018/Find pages that were visited in April 2018
h +3/Find pages that were visited from 3th of the current month up to now
h +3/ -10/Find pages that were visited from 3th and before 10th of the current month
h +/6/Find pages that were visited from June of the current year up to now
h =/5/ =/9/Find pages that were visited in May or September of the current year
h +6: -12:Find pages that were visited today from 6 am to 12 pm

After filters

Filters that will remove unwanted entries from the resulting set

FormExplanation
+url terms...include only pages with a URL that contain such terms
-url terms...exclude pages that contain a URL with such terms
+title terms...include only pages with a title that contain such terms
-title terms...exclude pages that contain a title with such terms

Example queryExplanation
h word -title wordFind pages containing 'word' only in URL
h word1 +url word2 word3Find pages containing 'word1' in either URL or title AND ALSO have 'word2' AND 'word3' in their URL

Sorting

FormExplanation
>u terms...
>url terms... 		sort result by url in ascending order
<u terms...
<url terms...		sort result by url in descending order
>t terms...
>title terms...		sort result by title in ascending order
<t terms...
<title terms...		sort result by title in descending order
>d terms...
>date terms...		sort result by date in ascending order
<d terms...
<date terms...		sort result by date in descending order
>v terms...
>visit terms...		sort result by visit count in ascending order
<v terms...
<visit terms...		sort result by visit count in descending order

Complex examples

Example queryExplanation
h word !tagFind pages containing 'word' in either URL or title and having 'tag' tag
h !tag1 !tag2 =/7/Find pages having 'tag1' and 'tag2' tags visited in July
h word +title word >urlFind pages containing 'word' in title and sort them by URL
h word1 word2 !tag1 !tag2 >tFind pages containing 'word1' and 'word2' in either URL or title, and having 'tag1' and 'tag2' tags, and then sort them by title
h word =1/ =8/ =15/Find pages containing 'word' in either URL or title that were visited on 1th, 8th or 15th of the current month
